# SOC Incidents

> Automated incident tickets generated by Log File Analyzer

This repository stores security incidents automatically created by the monitoring workflow.
Each issue represents a detected attacker and contains:

- IP address
- Severity level
- Local behavior analysis
- Global reputation intelligence
- Automated response taken
- First/last seen timestamps
---

## Purpose

This repository simulates a real SOC case management system.

Monitoring pipeline creates structured security cases that analysts can triage and investigate.

This mirrors how Jira / ServiceNow / SIEM ticket queues operate.
